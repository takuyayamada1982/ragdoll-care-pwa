# Ragdoll Care MVP Blueprint

## 目的

健康管理を堅苦しい入力作業にせず、日々の写真と観察を積み上げることで、家族が自然に変化へ気づける猫ログPWAにする。

## 初期スコープ

- スマホ専用UI
- 最初は1匹、設計は多頭飼い対応
- 最初は2人、将来3から5人対応
- ファミリーIDに複数ユーザーを紐づける
- 医療診断ではなく記録中心
- 通知は1日2回程度
- 写真は日常保存、アップロード時に圧縮
- AI要約や写真検索は後日拡張

## 記録項目

| 種別 | 主な項目 | MVP優先度 |
| --- | --- | --- |
| ごはん | 状態、カロリー、種類、メモ | 高 |
| 水 | 状態、量、メモ | 高 |
| トイレ | おしっこ、うんち、状態、回数、メモ | 高 |
| 体調 | 元気、眠そう、嘔吐、くしゃみ、目やに、毛玉 | 高 |
| 行動 | 睡眠、遊び、甘え、隠れる、鳴き方 | 中 |
| ケア | ブラッシング、爪切り、歯みがき、薬、通院 | 高 |
| 体重 | kg、測定日、メモ | 高 |
| 写真 | 日付、タグ、関連ログ、メモ | 高 |

## データモデル

- Family: 家族単位。共有ID、通知設定を持つ。
- Member: 家族メンバー。実運用では個別ログインと権限を持つ。
- Cat: 猫プロフィール。多頭飼い対応の中心。
- LogEntry: すべての観察ログ。猫、記録者、日時、種別、タグ、数値、メモを持つ。
- PhotoAsset: 圧縮画像、サムネイル、撮影日、タグ、関連ログを持つ。
- DailySummary: 将来の集計テーブル。AI要約やレポート生成の土台。

## 後日拡張

- AIによる週次要約
- 自然文検索
- 写真から状態タグ候補
- 通院前レポート
- 家族招待リンク
- Push通知
- オリジナル写真保存オプション
